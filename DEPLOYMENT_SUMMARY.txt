────────────────────────────────────────────────────────────────────────────

================================================================================
COMMON MISTAKES TO AVOID
================================================================================

❌ DON'T skip database table creation
   ✅ Always execute setup-database.sql first

❌ DON'T forget to create .env.local file
   ✅ Create it in project root with all 4 variables

❌ DON'T copy env values with quotation marks
   ✅ Values should NOT have quotes around them

❌ DON'T close terminal while server is running
   ✅ Keep terminal open during development

❌ DON'T use npm/yarn if you installed pnpm
   ✅ Stick with one package manager

❌ DON'T commit .env.local to Git
   ✅ Add to .gitignore before committing

❌ DON'T test without confirming email first
   ✅ Always click confirmation link in email

================================================================================
IF SOMETHING GOES WRONG
================================================================================

ISSUE: "Cannot find module"
FIX: rm -rf node_modules && pnpm install

ISSUE: "Database connection failed"
FIX: Verify .env.local has correct values

ISSUE: "Port 3000 already in use"
FIX: pnpm dev -- -p 3001 (use different port)

ISSUE: "Tables don't exist"
FIX: Re-run setup-database.sql in Supabase

ISSUE: "Can't log in"
FIX: Verify email is confirmed

ISSUE: "Dashboard is blank"
FIX: Check console (F12) for error messages

For more help: See DEPLOYMENT_GUIDE.md troubleshooting section
