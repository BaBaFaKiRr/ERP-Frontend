# Cursor Prompt — Build LEJER Communication Service

You are a senior staff software engineer tasked with designing and implementing a production-ready communication service for LEJER ERP.

## Background

LEJER is a multi-tenant ERP built with:

* Next.js (App Router)
* Node.js backend
* PostgreSQL (Supabase)
* Supabase Authentication
* Existing Organization → Users → Roles architecture

Every record in the database belongs to an organization via `organization_id`.

This communication module must **not** be built as a standalone chat application. Instead, it should be a first-class ERP service tightly integrated with every business object.

The goal is to create something similar to Slack + Microsoft Teams + Linear comments + Notion discussions.

---

Create a Communication Service that supports:

* Direct Messages
* Group Chats

# Unified Conversation Model

Instead of separate implementations, everything should use one Conversation model.

Conversation Types:

* direct
* group
* channel
* object_thread

An object discussion is simply another conversation with metadata:

```
object_type

object_id
```

This allows replies, search, notifications, mentions, attachments, etc. to work everywhere.

# Mention System

Implement a Slack-like mention system.

Typing:

```
@
```

opens a searchable user menu.

Example:

```
@John
```

Typing:

```
#
```

opens a searchable ERP object picker.

Example:

```
#SO-2028-9999
```

Supported object types:

Sales Orders

Purchase Orders

Invoices

Quotations

Customers

Vendors

Inventory Items

Employees

Projects

Tasks

Payments

Ledgers

Production Orders

Stock Transfers

Typing should immediately show autocomplete suggestions.

The dropdown should support keyboard navigation.

Selecting an item inserts a structured mention into the editor.

Mentions must not be stored as plain text.

Store structured metadata similar to:

```
{
  type: "user",
  id: "...",
  display: "John Doe"
}
```

or

```
{
  type: "sales_order",
  id: "...",
  display: "SO-2028-9999"
}
```

Messages should render mentions as clickable chips.

Clicking a mention should navigate directly to the referenced ERP record.

Hovering should show a small preview card.

# Code Quality

* Use TypeScript everywhere.
* Follow existing project architecture and coding conventions.
* Keep business logic separated from UI.
* Write reusable hooks and services.
* Use optimistic UI updates where appropriate.
* Handle loading, offline, retry, and error states gracefully.
* Build for scalability (millions of messages and thousands of organizations).
* Avoid N+1 queries.
* Add appropriate indexes to all messaging tables.
* Use cursor-based pagination for message history.
* Ensure APIs and components are well documented with concise comments where needed.
